# Brief — "Save for later" confirmation toast

A worked example used to prove the Designpowers pipeline runs end to end. Small on
purpose, so the whole studio fits in one readable artefact.

- **Problem:** When a user saves an article to read later, nothing confirms it
  worked, so they re-tap and second-guess.
- **Key task:** A user taps "Save for later" and needs immediate, legible
  confirmation it succeeded.
- **Users:** Full ability spectrum assumed — including low-vision users and people
  reading on a phone in bright sunlight (situational low contrast).
- **Quality bar:** Shipping component for a reading app; must pass WCAG AA.
- **Deliverable:** One confirmation toast component (HTML/CSS).

> Judgment vs. truth, declared up front: the strategy/copy/craft below are
> **judgment** (a model performing the skills). The contrast verdicts are **truth**
> — measured by the `designpowers-accessibility` MCP tool against the *actual* colours
> in the build, never estimated.
